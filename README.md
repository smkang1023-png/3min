# content-engine — 소재 1개 → 3포맷(블로그/뉴스카드/스레드) 콘텐츠 엔진

꾸준한 수익형 자동화의 뼈대. 소재 수집부터 포맷별 출력까지 한 흐름으로 돌아간다.

## 파이프라인

```
trends.py ──→ (Claude가 콘텐츠 JSON 작성) ──→ render_cards.py  → 인스타 캐러셀 PNG
   실측 수집        content/*.json          ├→ make_threads.py  → 스레드 체인 txt
                                            └→ blog_draft.md    → 네이버 블로그 초안
```

- `trends.py` — 구글 트렌드(KR)·구글 뉴스(전체/경제/기술) RSS 실측 수집 → `data/topics_*.json`
- `content/*.json` — 하루치 콘텐츠 원본(단일 소스). Claude Code 세션/예약 작업에서 작성
- `render_cards.py` — 1080×1350 인스타 캐러셀 (표지 + 본문 N장 + 아웃트로, 페이지 도트/핸들 포함)
- `make_threads.py` — 스레드 체인(훅+답글) 텍스트, 게시물당 500자 검사
- 블로그 초안 — 같은 JSON을 확장해 SEO 구조의 .md로 (Claude가 작성)

## 일일 운영 루프 (목표)

1. 아침: Claude Code 예약 작업이 `trends.py` 실행 → 소재 선정 → 콘텐츠 JSON 작성 → 3포맷 렌더링
2. 사용자: 출력물 검수(5분) → 인스타/스레드/블로그에 게시
3. 주간: 계정 인사이트 수치를 피드백해서 훅/포맷 조정

## 자동 게시 (다음 조각)

- **스레드**: Meta 공식 Threads API로 자동 게시 가능 (Meta 개발자 앱 + 액세스 토큰 필요)
- **인스타그램**: 비즈니스 계정 전환 후 콘텐츠 게시 API 사용 가능
- **네이버 블로그**: 공식 글쓰기 API 없음 → 초안 복사-붙여넣기(비공식 자동화는 저품질 위험이라 안 함)

## 원칙

- 정보성 콘텐츠는 헤드라인/공식 통계 범위 안에서만 서술 (추측·과장 금지)
- 투자 정보에는 면책 문구 필수
- 같은 템플릿의 완전 복제 양산 금지 — 구조는 유지, 문장과 소재는 매일 다르게
