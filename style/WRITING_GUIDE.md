# 글쓰기 에이전트 작성 규칙

콘텐츠 JSON·스레드·블로그를 작성하는 모든 세션은 이 문서를 따른다.
이 문서는 살아있는 문서다 — 주간 인사이트 수치를 근거로 훅 패턴을 추가/삭제한다.

## 1. 공통 원칙 (모든 플랫폼)

- **한 문단 = 한 생각.** 문단은 2~3문장에서 끊는다. 문단 사이에 반드시 빈 줄.
- **숫자가 먼저 온다.** "많이 올랐다" 금지 → "3.2% 올랐다". 숫자 없는 문장이 연속 2개를 넘기지 않는다.
- **사실 → 체감 순서.** 먼저 무슨 일이 있었는지(헤드라인 범위 내 사실), 그다음 "그래서 나한테 뭐가 달라지는지" 한 문장.
- **추측 금지.** 헤드라인·공식 통계에 없는 내용은 쓰지 않는다. 전망을 전할 땐 반드시 출처 주체를 밝힌다("~라는 전망이 나왔습니다").
- 투자 관련 내용이 있으면 면책 문구 필수.

### 법적 안전 규칙 (정보통신망법 2026-07-07 개정 대응 — 위반 시 발행 금지)

- **검증 매체만 근거로.** topics_*.json에서 `trusted: true` 항목만 사실 근거로 쓴다. 포털 링크(v.daum.net 등)·무명 매체 단독 보도는 소재로 쓰지 않거나, 검증 매체의 같은 보도를 찾아 근거로 삼는다.
- **source 필드 필수.** 모든 카드에 근거 기사(outlet/title/link)를 기록한다. 렌더러가 카드에 "자료: ○○ 보도"로 표기한다.
- **시제 단정 금지.** 계획·발표는 발표로 쓴다: "감원했다"✕ → "감원 계획을 밝혔다"○. 아직 일어나지 않은 일을 완료형으로 쓰면 그 자체가 허위가 된다.
- **의혹·수사·재판은 인용형으로만.** "담합했다"✕ → "검찰이 담합 정황을 적발했다고 밝혔다"○. 발표·보도 주체를 반드시 명시.
- **숫자는 기사에 있는 그대로.** 계산·추정·어림값 생성 금지.
- **개인·기업 평가 금지.** 사실 전달만 하고 조롱·비방·부정적 낙인 표현은 쓰지 않는다.

## 2. AI 티 제거 블랙리스트

아래 표현·습관이 하나라도 있으면 그 문장은 다시 쓴다.

| 금지 | 대신 |
|---|---|
| "~에 대해 알아보겠습니다" | 그냥 본론부터 시작 |
| "~하는 것이 좋습니다" | "~하세요" 또는 구체적 행동 |
| "결론적으로", "요약하자면" | 소제목이나 그냥 결론 문장 |
| "다양한", "효과적인", "중요한" (수식어 남용) | 구체적 명사·숫자로 대체 |
| "~라고 할 수 있습니다" | "~입니다" |
| 모든 문장이 비슷한 길이 | 짧은 문장을 섞는다. "이게 큽니다." "왜냐고요?" |
| 완벽한 문어체 일변도 | 구어 한 스푼: "장바구니가 무거운 게 기분 탓이 아니었습니다" |
| 이모지 남발 | 스레드 훅에 최대 1개, 블로그 본문 0개 |
| 번호·불릿의 기계적 반복 | 카드/문단마다 문장 구조를 바꾼다 |

## 3. 훅 공식 (실측 기반 — 주간 업데이트 대상)

수집 데이터에서 검증된 패턴. 매일 하나를 고르되 **어제와 같은 공식을 연속 사용하지 않는다.**

1. **손해회피형** — "안 사면 손해", "모르면 나만 손해", "품절 대란" (다이소/꿀템 니치 실측 상위)
2. **숫자 반전형** — "1억 넣었으면 2억 됐다", "3조 원이 강제로 팔렸다"
3. **오늘변화형** — "오늘부터 바뀝니다", "이제 안면인증까지" (제도·요금 소재에 최적)
4. **대상저격형** — "전세 구하는 분들은 오늘 꼭", "완속 충전 쓰는 사람만 유리"
5. **질문형** — "핸들도 안 잡았는데 운전자 책임?" (사용 빈도 낮게, 남발 시 어그로 판정)

## 4. 플랫폼별 구조

### 인스타 뉴스카드 (콘텐츠 JSON)
- 카드 제목: 형광펜에 얹히는 글자. **총 22자 이내** (렌더러가 11자/줄로 자동 줄바꿈, 2줄 초과 시 넘침), 숫자나 반전 단어 포함.
- **emoji 필드 필수**: cover·각 card·outro에 소재를 직관적으로 보여주는 이모지 1개 (렌더러 v3가 카드 우상단 아이콘으로 그림). 같은 날 카드끼리 중복 금지, 소재와 무관한 장식용 금지.
- 카드 본문: 정확히 2~3문장 — ① 사실 ② (필요시) 배경 한 줄 ③ 체감 포인트.
- 카드 한 장은 15초 안에 읽혀야 한다. 본문 90자 초과 금지.
- 커버 소제목: 그날 소재 3~4개를 " · "로 나열 + 행동 유도 반 줄.

### 스레드
- **첫 15자가 전부다.** 첫 줄에 숫자·반전·대상 중 하나가 반드시 들어간다.
- 문장마다 줄바꿈. 3~4줄마다 빈 줄로 호흡.
- 게시물당 500자 이내(도구가 검사함). 체인은 훅 1 + 본문 3~4 + 마무리 1.
- 마무리는 **대화 유도 질문**으로 끝낸다: "여러분은 완속파세요, 급속파세요?"
- 체인 외에 **단문 2~3개**를 별도로 뽑는다: 그날 가장 자극적인 사실 하나 + 질문 한 줄짜리.

### 블로그 (SEO)
- 제목 공식: `핵심 키워드 + 숫자 + 연도/날짜` — "전기차 충전요금 5단계 개편, 뭐가 달라지나 (2026년 7월)"
- 도입 3줄 안에 결론이 나온다. "알아보겠습니다"류 워밍업 금지.
- 소제목(##)마다: 사실 문단 → 해설/체감 문단. 두 문단을 넘기지 않는다.
- 굵게(**)는 숫자와 핵심 명사에만. 한 소제목당 최대 2회.
- 마지막: "오늘의 한 줄 정리" + 면책·출처.

## 5. 주간 업데이트 절차

- 저장/팔로우 상위 게시물의 훅을 §3에 패턴으로 추가
- 2주 연속 하위인 패턴은 삭제
